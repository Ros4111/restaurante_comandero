PROYECTO: Sistema de Pedidos para Restaurante (Flutter + Raspberry Pi + Sunmi V2 + PHP)
0 Objetivo
Vas a diseñar una aplicación completa para un restaurante donde los camareros toman pedidos de los clientes con un teléfono con Android 7.1.2 (Sunmi V2 que incorpora una impresora térmica). Los camareros tomarán nota de los productos solicitados. Toda la información se guardará en un servidor web con Apache y Mysql. Cada nuevo pedido se imprimirá en el teléfono del camarero (para entregar al cliente como prueba de que todo se hizo correctamente) y también será imprimido en la zona de preparación correspondiente (por ejemplo, la bebida será enviada a la impresora instalada en la barra y la comida a la impresora instalada en la cocina. Un producto solo se puede imprimir en una impresora).
También debes crear el código PHP que se ejecutará en el servidor Web para gestionar toda la información entre la base de datos y los teléfonos Sunmi V2

Debes crear el código para imprimir (con cola de impresión intermedia) en las impresoras de tickets ESC/POS desde la Raspberry Pi.
Habrá un sistema de bloqueo para que solo 1 usuario modifique una mesa, basado en un campo de hora.

Cuando el camarero cancela un producto ya guardado en otra sesión, se envía a la impresora asignada un mensaje con el nombre del camarero, hora, cantidad y nombre del producto. 
No es necesario ver el histórico de pedidos de mesas ya cerradas porque se cobraron.
Se estima que el número máximo de mesas abiertas simultáneamente serán 50.
Si cae el servidor o el wifi, la aplicación se queda esperando poder volver a conectar mientras espera un mensaje en pantalla “	servidor inaccesible” en la parte superior de la pantalla, permitiendo seguir editando la mesa para sincronizar después.

1️⃣ Arquitectura General del Sistema
Hardware
•	Router WiFi (red local interna) 
•	5 × Dispositivos Sunmi V2 (Android 7.1.2) con impresora térmica integrada 
•	2 × Impresoras de tickets ESC/POS por TCP/IP (conectadas por cable al router) con IP fija asignada
•	1 × Raspberry Pi 4 con: 
o	IP fija 
o	Apache 
o	PHP 
o	MySQL 
o	Certificado HTTPS válido 
•	Monitor HDMI + teclado + ratón conectados a la Raspberry

•	Autenticación mediante JWT con duración de 24 h
•	La Raspberry es: 
o	Servidor API REST 
o	Gestor de impresión de barra y cocina 
o	Fuente única de verdad 
•	Los móviles: 
o	Solo gestionan UI y captura de pedidos 
o	Imprimen ticket de confirmación en su impresora Sunmi 
o	Nunca imprimen en barra/cocina directamente

1.	Se busca archivo local de configuración con IP del servidor. 
2.	Si está vacío: 
o	Pantalla para introducir IP 
o	Botón “Guardar” 
o	Solo guarda si conexión HTTPS exitosa 
3.	Si existe IP: 
o	Solicita lista de usuarios al servidor 
o	Si falla conexión tras 3 intentos: 
	Mensaje: “Imposible conectar con el servidor” 
	Botones: Reintentar / Configurar

7️⃣ Sistema de Bloqueo de Mesa
Cuando un camarero entra:
•	Se bloquea inmediatamente Pedido_cabecera
•	Se actualiza: 
o	Id_usuario_bloqueo 
o	Hora_bloqueo 
Si otro camarero entra:
•	Puede ver pedido pero:
o	Modo solo lectura 
o	Mensaje: “Mesa bloqueada por X usuario” 
Cada minuto:
•	Se envía ping al servidor desde el camarero que abrió primero la mesa
•	Se actualiza Hora_bloqueo para que pasados 3 minutos no se desbloquee automaticamente
Si pasan 3 minutos sin ping:
•	Mesa queda automáticamente desbloqueada
En el servidor se comprobará que los permisos para realizar algún cambio en una mesa no han caducado, por si el teléfono perdió la conexión Wifi por más de 3 minutos y otro usuario adquirió los permisos para realizar cambios.
Un super usuario tiene permisos para expulsar a otro usuario de una mesa

1️⃣1Guardado y Cierre
Al cerrar pantalla:
•	Se envían todos los productos al servidor 
•	En el servidor: 
o	Detecta cambios 
o	Genera Registro_de_cambios 
o	Envía impresión a barra/cocina 
o	Marca Impreso_si_no = 1 después de Conexión exitosa + envío sin excepción 
Nunca se reimprime producto ya impreso sin cambios.
Para detectar los cambios, el servidor comparará todas las líneas del pedido con la base de datos gracias al campo Id_linea. Localizada la misma línea de la base de datos, comparará si se borró, se cambió cantidad o el comentario. Después de guardar los cambios, se imprimirá en la impresora correspondiente.
